TAREFA 1 — Adicionar campos de perfil na tabela users
No arquivo shared/schema.ts, adicionar os seguintes campos opcionais 
à tabela users existente (NÃO recriar a tabela, apenas adicionar campos):

  fullName       text (nullable)
  cpfCnpj        text (nullable)  
  phone          text (nullable)
  addressStreet  text (nullable)
  addressNumber  text (nullable)
  addressComplement text (nullable)
  addressNeighborhood text (nullable)
  addressCity    text (nullable)
  addressState   text (nullable)
  addressZip     text (nullable)

Todos os campos devem ser nullable para não quebrar usuários existentes.

TAREFA 4 — Adicionar métodos no storage
No arquivo server/storage.ts, adicionar dois novos métodos na 
interface e na implementação:

Método 1: getUserProfile(userId: number)
  - Busca o usuário pelo id
  - Retorna: id, username, email, fullName, cpfCnpj, phone,
    addressStreet, addressNumber, addressComplement,
    addressNeighborhood, addressCity, addressState, addressZip
  - NÃO retornar: password, sessionTokens ou dados sensíveis

Método 2: updateUserProfile(userId: number, data: Partial<UpdateUserProfile>)
  - Atualiza apenas os campos fornecidos (partial update)
  - Retorna o usuário atualizado (sem senha)
  - Usar o schema updateUserProfileSchema para validação

TAREFA 5 — Criar endpoints de perfil
No arquivo server/routes.ts, adicionar dois novos endpoints 
na seção após os routes de auth existentes:

Endpoint 1: GET /api/users/profile
  - Requer autenticação (usar middleware requireAuth existente)
  - Chama storage.getUserProfile(req.user.id)
  - Retorna os dados do perfil sem senha
  - Cache-Control: private, no-cache (dados pessoais)

Endpoint 2: PATCH /api/users/profile
  - Requer autenticação (usar middleware requireAuth existente)
  - Valida o body com updateUserProfileSchema
  - Chama storage.updateUserProfile(req.user.id, validatedData)
  - Retorna o perfil atualizado
  - Registrar evento de auditoria com logAuditEvent:
    ação: 'profile_update', incluir no details quais campos foram 
    alterados (apenas as chaves, não os valores)
